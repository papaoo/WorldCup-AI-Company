# 我让 Codex 搭了一个“世界杯 AI 公司”，第一场就预测对了

昨晚世界杯第一场，墨西哥 2:0 南非。

如果只是看球，这是一场带着红牌、主场声浪、揭幕战情绪的比赛。

但对我来说，这场比赛还有另一层意义：

**我用 Codex 辅助开发的“世界杯 AI 公司”，第一场预测命中了。**

不是 Excel 表格，不是手写几条规则，也不是让大模型拍脑袋说一句“我觉得墨西哥会赢”。

而是一个真正跑起来的系统。

它会自动抓公开数据源，读取 FIFA 排名、Elo 战力、近期国际比赛结果、赛程状态、新闻情报；它会把每支球队分配给一个 AI 员工；它会生成球队研究报告、赛前胜率、投注风险提示；比赛结束后，它还会把比分写回系统，生成复盘记录，计算预测是否命中。

第一场，系统给墨西哥的赛前胜率约为 65.7%。

结果：墨西哥 2:0 南非。

**方向命中。**

当然，我不会因为一场比赛就说“AI 已经封神”。一场样本没有统计意义。但它至少证明了一件事：

**这个系统不是玩具页面，它已经形成了一个可运行的预测闭环。**

数据采集 -> 结构化建模 -> AI 员工研判 -> 前端展示 -> 比赛结束 -> 自动复盘 -> 进入记忆系统。

这套东西最让我兴奋的，不是预测对了一场，而是开发方式本身变了。

过去做这样一个系统，需要产品经理、后端、前端、数据工程、算法、测试一起排期。现在，我把需求不断讲给 Codex，它读代码、改代码、跑测试、修 bug、接数据源、做回归。

我们一边讨论，一边把系统从“一人公司”改造成“世界杯 AI 公司”。

最关键的是，它不是一次性生成代码就结束。

我让它必须做回测，必须做日志，必须记录数据来源，必须区分事实、新闻、模型判断和大模型分析。它不能只输出一段漂亮文案，它要告诉我：数据从哪里来，模型怎么算，为什么给这个胜率，赛后到底准不准。

目前系统的历史回测结果并不完美。

最近一年 80 个可映射样本，Top1 命中率约 56.25%；强倾向场次命中率约 68.18%；但平局识别很弱，只有约 5% 的召回。

这反而让我更放心。

因为它没有伪装成神棍。

它知道自己哪里强，哪里弱，哪里缺数据。

现在最大的问题是：伤停、首发、临场赔率、真实赛中事件还没有全部稳定接入。新闻 RSS 只能作为情报线索，不能直接当事实。大模型负责解释、分拣和复核，但不能替代结构化数据模型。

这就是我想做的 AI 产品哲学：

**大模型不是算命先生，它应该是一个组织里的员工。**

数据源是情报员，预测模型是分析师，LLM 是研究员和审计员，CEO Agent 负责综合判断，而系统日志和回测是公司的财务报表。

没有日志，就没有信任。

没有回测，就没有进步。

没有数据来源，就没有预测资格。

所以这不是一个“AI 猜球系统”。

它更像一个自动运转的小型 AI 研究公司。

## 这个系统到底强在哪

### 1. 自动采集，而不是手动喂数

系统会定时抓公开数据源，自动同步赛程、比分、排名、Elo、新闻情报和赛果状态，不需要我每次手动导入。

### 2. 每支球队都有“专属员工”

不是一个大模型把所有球队一锅端，而是每支球队都对应一个研究员、风险员、CEO 决策视角。这样每场比赛都有自己的分析链路。

### 3. 胜率不是拍脑袋

系统不是只看一条新闻，而是同时参考 FIFA 排名、Elo、近期战绩、主办国因素、证据包和记忆系统，再把这些信号合成胜率。

### 4. 比赛结束后会自动复盘

一旦比分回写，系统会自动生成复盘记录、命中情况和记忆，后面还能进入准确率统计与策略评估。

### 5. 有日志、有证据、有回测

每一次数据来源、每一次 LLM 交互、每一次研报生成、每一次预测结果，都会进入日志和证据链。它不是黑箱，是真正可审计的系统。

### 6. 前端不是死表格

现在的界面已经有预测队列、球队研究室、数据可信度、研报归档、赛事进程和比分展示，适合做截图讲故事，也适合以后继续重构成更像“经营模拟”的风格。

墨西哥 2:0 南非，只是它的第一张成绩单。

后面还有 100 多场比赛。每一场都会进入系统，每一场都会更新模型，每一次错误都会变成记忆，每一次命中都会接受复盘。

如果这套系统最终能跑完整届世界杯，那它证明的也许不只是“能不能预测足球”。

它证明的是：

**一个普通人，借助 Codex 和 AI Agent，已经可以独立搭建过去需要一个小团队才能完成的复杂系统。**

这才是我真正兴奋的地方。

## 可用小标题

### 1. 第一场：墨西哥 2:0 南非

这场比赛给了系统第一个真实样本。系统赛前判断墨西哥占优，实际结果也是墨西哥取胜。

### 2. 不是猜球，而是工程系统

系统不是一句“我觉得谁会赢”，而是通过数据源、模型、AI 员工、日志和回测组成完整链路。

### 3. Codex 改变了开发方式

Codex 不只是生成代码，而是参与架构、数据源接入、后端开发、前端展示、测试回归和问题修复。

### 4. 当前模型还不完美

历史回测显示模型对强倾向场次更有效，但平局识别仍然偏弱。这个缺陷会成为后续优化重点。

### 5. AI Agent 的真正价值

大模型不是一个答案机器，而是系统里的一个岗位。它负责解释、分拣、审计和总结，而不是替代全部事实判断。
